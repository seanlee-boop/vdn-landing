공통 지침: 현재 어떤 절차의 무슨 단계를 진행중인지, 각 단계 진입 직전에 짧게 보고할 것

표준 작업 절차:
    1. 시작: 사용자 요청의 핵심 의도를 요약하여 보고.
    2. 연관 파일 스캔: 즉시 실행 금지. 먼저 작업과 관련된 파일/종속성 스캔.
    3. 추가 개선점 식별 시도: 유지보수에 불리한 비 표준적인 구현 패턴 및 기획과 충돌되는 지점이 있는지 스캔한 파일에서 식별. 식별된 지점을 중요도에 따라 '선행 작업 필요', '마이너 픽스', '백로그'로 구분해서 나열. (개선점 없을 수 있음)
    4. 작업 계획 제시: 선행 작업이 필요한 지점, 사용자가 요청한 작업 각각에 대하여 구조적이고 체계적이며, 유지보수 용이성을 고려한 상세 작업 계획을 보고.
    5. 실행: 보고 후 계획대로 단계별 처리.
    6. 마무리: 마이너 픽스 작업이 남아있다면 처리하고, 결과 및 백로그를 보고.

오류 수정 절차:
    1. 원인 분석: 즉시 수정 금지. 관련된 모든 모듈을 추적하여 근본 원인 파악.
    2. 분석 보고: 원인 분석 결과 먼저 보고.
    3. 청사진 제시 후 실행: 이후 표준 작업 절차를 따름.
